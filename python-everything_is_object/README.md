# Python - Everything is Object

## Overview
This project is an intensive exploration of Python's memory management, focusing on the distinction between **mutable** and **immutable** objects. It covers how Python handles variables as references and the implications of this behavior on memory and function calls.

## Learning Objectives
* Understand object identity (`id()`) and types (`type()`).
* Differentiate between mutable (lists, dicts) and immutable (ints, strings, tuples) types.
* Grasp the concepts of aliasing and "Call by Object Reference".

## Contents
* **Tasks 0-28:** Quick-fire questions and answers about Python's object behavior (stored as `.txt` files).
* **Task 29:** A comprehensive technical blog post explaining these concepts in depth.
